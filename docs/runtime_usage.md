# Sisal Runtime and C Code Generation Usage

This document explains how to compile and run C/C++ code generated by the Sisal compiler using the provided runtime.

## 1. Prerequisites
- **Compiler**: `clang++` (recommended) or `g++`.
- **Operating System**: macOS (for Apple Silicon optimizations and GCD support).
- **Frameworks**: `Accelerate.framework` (on macOS).

## 2. Generating C/C++ Code
Use the Sisal compiler with the `--c` flag to generate C++ code from a Sisal source file:

```bash
./sisal_compiler --c=output.cpp input.sis
```

To enable Rank-Agnostic Dope Vectors (DV), add the `--dv` flag:

```bash
./sisal_compiler --dv --c=output.cpp input.sis
```

## 3. The Sisal Runtime
The runtime consists of a single header-only file: `runtime/sisal_runtime.h`. This file provides:
- The `sisal_array_t` structure (Dope Vector).
- Memory allocation functions (`sisal_alloc_unified`).
- Array manipulation and reduction functions.
- Math helpers.
- Parallel loop orchestration using Grand Central Dispatch (GCD).

## 4. Compiling the Generated Code
To compile the generated code, you need to:
1. Include the directory containing `sisal_runtime.h`.
2. Link with the `Accelerate` framework (for vDSP/AMX acceleration).

Example command:
```bash
clang++ -O3 output.cpp main_harness.cpp -I./runtime -framework Accelerate -o my_program
```

*Note: `main_harness.cpp` is your own entry point that calls the generated Sisal functions.*

## 5. Directory Structure
- `runtime/`: Contains the header-only runtime implementation.
- `test/silicon/`: Contains regression tests and sample test harnesses.

## 6. Portability Note
The current runtime implementation uses macOS-specific headers (`dispatch/dispatch.h`, `Accelerate/Accelerate.h`). To run on other platforms, these dependencies would need to be abstracted or replaced with standard C++ threads/OpenBLAS.
